# Discrete Math Toolkit - Project Plan

## Problem Definition

Students often learn discrete mathematics through textbooks and lectures but struggle to understand how the concepts are implemented in software.

Existing libraries provide solutions but rarely explain how those solutions are derived.

This project aims to bridge the gap between theory and implementation by providing educational implementations of core discrete mathematics concepts.

---

# Objectives

## Primary Objectives

* Learn discrete mathematics through implementation.
* Demonstrate clean Java software design.
* Build algorithms from first principles.
* Create reusable educational examples.

## Secondary Objectives

* Practice object-oriented design.
* Practice testing and documentation.
* Serve as a portfolio project.

# Functional Requirements

FR-1 Convert values between bases 2–36.

FR-2 Determine whether a number is prime.

FR-3 Perform set operations.

FR-4 Generate truth tables.

FR-5 Compute permutations and combinations.

FR-6 Analyze mathematical relations.

FR-7 Perform graph traversals.

FR-8 Display optional step-by-step execution details.

---

# Non-Functional Requirements

## Maintainability

Modules should be loosely coupled and highly cohesive.

## Reliability

Invalid input must be handled gracefully.

## Testability

Business logic must be independently testable.

## Portability

Application must run on all major operating systems supporting Java.
